# UI 风格对齐文档

## 设计关键词

- 精致。
- 紧凑。
- 工具型。
- 清晰。
- 可诊断。
- Minecraft 轻量暗示。

界面应该像一个认真做事的桌面工具，而不是宣传页。第一屏直接进入可操作界面。

## 视觉方向

主题名：

```text
Warm Pixel Workbench
```

核心感觉：

- 浅色工作台。
- 高信息密度。
- 小面积像素纹理。
- 绿色表示可执行和成功。
- 铜色表示注意、冲突、成本和待确认。
- 红色只表示真实错误。

## 色彩

| 用途 | 色值 | 说明 |
| --- | --- | --- |
| 背景 | `#F7F4EC` | 温暖浅底 |
| 面板 | `#FFFFFF` | 主要内容区域 |
| 弱面板 | `#EFEADF` | 次级区域 |
| 主文本 | `#1E1D1A` | 接近墨黑 |
| 次文本 | `#6B665D` | 辅助信息 |
| 分割线 | `#DED8CC` | 卡片边框 |
| 主色 | `#1F8A5B` | 开始、成功、进度 |
| 主色 hover | `#166B48` | 主按钮 hover |
| 强调 | `#B87932` | 警告、冲突、成本 |
| 错误 | `#B8423A` | 错误 |
| 信息 | `#4F6F73` | 低优先级状态 |

避免：

- 大面积紫色。
- 大面积蓝紫渐变。
- 深色主题作为默认。
- 米色单色铺满导致无层级。

## 字体

推荐：

- UI 中文：`Microsoft YaHei UI` 或 `Noto Sans SC`。
- 数字和代码：`JetBrains Mono`。

字号：

| 场景 | 尺寸 |
| --- | --- |
| 页面标题 | 22 |
| 分区标题 | 16 |
| 表格正文 | 13 |
| 辅助文本 | 12 |
| 状态徽标 | 12 |
| 数字指标 | 20 |

不使用随窗口宽度缩放的字体。

## 布局

基础尺寸：

- 桌面基准：`1365x768`。
- 左侧导航：`232px`。
- 顶部工具条：`56px`。
- 页面内边距：`20px`。
- 卡片圆角：`6px`。
- 表格行高：`38px`。
- 按钮高度：`32px` 或 `36px`。

页面结构：

```text
App Shell
  Sidebar
  Main
    Topbar
    Page Content
```

不要把页面大区块做成漂浮大卡片；卡片只用于指标、表格容器、详情面板和弹窗。

## 组件规范

### 按钮

- 图标按钮优先使用 lucide 图标。
- 主要动作使用绿色实心按钮。
- 危险动作只在确认弹窗中出现红色。
- 替换资源包这类动作必须二次确认。

### 表格

表格是核心组件，必须支持：

- 搜索。
- 排序。
- 状态徽标。
- 选中行详情。
- 横向滚动。
- 空状态。
- 错误状态。

### 进度

进度不能只有一个百分比。需要同时显示：

- 当前阶段。
- 已完成条目。
- 失败条目。
- ETA。
- 当前吞吐。
- 词典命中率。
- LLM pending 数。

### 日志

日志页必须可过滤：

- main。
- job。
- error。
- http。
- debug。
- validation。

错误详情要能关联到：

- modid。
- translation key。
- 原始文本。
- LLM request id。
- retry history。

## 页面清单

### 品牌与 logo

用途：

- app icon。
- 侧边栏 wordmark。
- 资源包 pack.png 可选基础。

图形方向：

- blocky `A`。
- 翻译气泡。
- 小方块。

### 总览页

必须看到：

- 当前实例路径。
- 扫描按钮。
- 统计卡片。
- 已识别资源包。
- 待翻译数量。
- 开始翻译按钮。

### 翻译进度页

必须看到：

- 阶段进度。
- per-mod 队列。
- 费用/token。
- ETA。
- 当前 batch 预览。
- 格式校验警告。

### 词典页

必须看到：

- 搜索和过滤。
- 来源优先级。
- 原文、译文、modid、key。
- 右侧纠错。
- 导入导出。

### 设置页

必须看到：

- Base URL。
- API Key。
- Model。
- 并发。
- batch size。
- 超时和重试。
- 资源包复用开关。
- FTB 开关。
- 日志策略。

### 打包确认页

必须看到：

- 生成资源包摘要。
- 资源包顺序。
- 冲突。
- dry-run diff。
- 复制/替换动作。

### 日志页

必须看到：

- 日志过滤。
- 错误详情。
- 导出日志。
- 日志保留策略。

### FTB 页

必须看到：

- quest tree。
- 条目表。
- 原文/译文预览。
- 手动纠错。

### 硬编码实验页

必须看到：

- 实验状态。
- 风险徽标。
- Vault Patcher 草案。
- 手动确认。
- 禁止自动应用。
