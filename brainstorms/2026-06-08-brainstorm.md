# Brainstorm Brief: Aaalice MC Translator 加载界面设计

## Current leaning

经过多轮讨论，形成了一个**基本完整的加载界面设计方案**，包含视觉构成、动画流水线、预加载契约和边界行为四个维度。用户在每个维度上都做出了明确选择，设计方向已经收敛。

## Design decisions reached

| 维度 | 决策 | 理由 |
|------|------|------|
| **视觉风格** | 极简品牌动画（扫描线渐现） | 呼应产品核心行为「扫描模组文本」，简洁专业 |
| **品牌图标** | 菱形书页（◇◆） | 书页 = 翻译 + 钻石 = Minecraft，双关设计契合产品身份 |
| **品牌标识** | 完整 logo + 副标题 + 图标 | 「Aaalice」+「Minecraft Translator」+ 品牌图形 |
| **色彩方案** | 跟随主题色 | 加载界面与后续 UI 无缝衔接 |
| **进度样式** | 分阶段确定进度条 + 阶段步骤条 | 用户清晰知道在等什么 |
| **动画节奏** | linear（扫描线）+ ease-out（显现） | 稳重专业，不喧宾夺主 |
| **预加载接口** | 单 command，通过 Tauri event 推送 4 阶段进度 | 前端管理简单，只需 listen 一个通道 |
| **最短展示** | 强制 800ms 最低展示 | 保证品牌露出，快速加载下用脉冲光效补齐 |
| **深色模式** | 完全跟随主题配色 | 视觉一致性好 |
| **过渡行为** | 平滑渐隐（400ms）→ 主界面淡入（300ms） | 过渡自然不突兀 |
| **无障碍** | 暂不处理 prefers-reduced-motion | 桌面用户需求度低 |

## 主要设计要点

**视觉布局**：菱形书页图标（48×48px）位于中央上方，下方品牌名「Aaalice」（纤细字重 36-40px），再下方副标题「Minecraft Translator」。一条 1-2px 主题色扫描线从左至右匀速移动，底部 4px 进度条 + 阶段步骤条显示「设置 ✓ → 本地 ✓ → 词典 ⟳ → LLM」四个阶段。

**预加载流水线**：设置(0-25%) → 本地元数据(25-50%) → 词典预热(50-75%) → LLM 模型列表(75-100%)。每阶段失败不打阻塞，标记 ⚠ 后继续。

**动画时序**：背景淡入(300ms) + 图标上浮(500ms) → 扫描线左端出发 → 品牌名逐字母显影(60ms/字母) → 副标题淡入(200ms) → 进度同步更新。
